# Translation Guide

Docker Manager uses the `easy_localization` package for internationalization. Translation files are stored as JSON in the `assets/i18n/` directory.

## How to Add a Translation

1. **Open an issue** to discuss the language you want to add
2. **Fork the repository** and create a branch: `git checkout -b add-LANGUAGE-translation`
3. **Copy the base file**: `cp assets/i18n/en-US.json assets/i18n/YOUR-LOCALE.json` (e.g., `fr.json`, `de.json`, `pt-BR.json`)
4. **Translate all values** in your new file (keep keys unchanged, only translate the values)
5. **Register your locale** in `lib/main.dart`:
   ```dart
   supportedLocales: const [
     Locale('en', 'US'), 
     Locale('es'),
     Locale('fr'),  // Add your locale here
   ],
   ```
6. **Add display name** in `lib/presentation/screens/settings_screen.dart` (search for the `switch (locale.languageCode)` block and add your language):
   ```dart
   case 'fr':
     displayName = 'Fran√ßais';  // Add your language display name
     break;
   ```
   Add it in **both** switch statements in the language dropdown section.
7. **Submit a Pull Request** linking to your discussion issue
8. **Test the APK** generated by GitHub Actions from the workflow artifacts
9. **Share your draft** in the issue for feedback from other native speakers

## Translation Guidelines

**Important:**
- Keep `{}` placeholders intact (e.g., `"Selected server: {}"`)
- Maintain the nested JSON structure
- Translate values only, not keys
- Use `\n` for line breaks
- Preserve technical terms where appropriate (Container, Docker, etc.)

**Structure:** Files use nested objects by feature (`app`, `common`, `connection`, `navigation`, `servers`, `containers`, `images`, `volumes`, `networks`, `system_info`, `settings`, `operations`, `actions`, `shell`)

**Example:**
```json
{
  "app": {
    "title": "Your Translation"
  },
  "common": {
    "cancel": "Your Translation",
    "delete": "Your Translation"
  }
}
```
